KickClip 개인정보처리방침

시행일: 2026년 4월 24일
최종 수정일: 2026년 4월 24일

KickClip(이하 "본 서비스")은 사용자의 개인정보를 중요하게 생각하며, 다음과 같이 개인정보를 수집, 이용, 보호합니다.

1. 수집하는 개인정보 항목

1.1 Google 계정 정보

Google OAuth 인증 시 다음 정보를 수집합니다:

  • 이메일 주소 (email)
  • 표시 이름 (displayName)
  • 프로필 사진 URL (photoURL)
  • Google 계정 고유 식별자 (UID)
  • 이메일 인증 여부 (emailVerified)
  • 인증 제공자 정보 (provider: google.com)

1.2 사용자 생성 콘텐츠

사용자가 명시적으로 저장하는 웹 콘텐츠:

  • 저장한 웹페이지의 URL
  • 웹페이지 제목, 설명, 썸네일 등 메타데이터
  • 사용자가 지정한 카테고리 및 폴더 정보
  • 저장 시각

1.3 기기 정보 (저장 위치 선택에 따라)

  • 사용자가 선택한 로컬 폴더 경로 정보 (브라우저 내부 저장, 외부 전송 없음)
  • 또는 사용자가 선택한 Google Drive 폴더 정보

2. 개인정보의 이용 목적

수집한 정보는 다음 목적으로만 사용됩니다:

  • 사용자 계정 인증 및 식별
  • 사용자별 데이터 분리 및 관리
  • 저장한 콘텐츠 조회 및 관리 기능 제공
  • 사용자가 선택한 저장 위치(로컬 폴더 또는 Google Drive)로의 파일 저장
  • 서비스 개선 및 오류 추적 (익명화된 로그)

3. 개인정보의 저장 및 보관

3.1 저장 위치

  • 사용자 프로필 및 메타데이터: Google Firebase Firestore (서버 위치: 아시아 북동부, 서울)
  • 파일 콘텐츠: 사용자가 선택한 로컬 폴더 또는 사용자의 개인 Google Drive
  • 임시 데이터: 브라우저 내 IndexedDB 및 chrome.storage

3.2 보관 기간

  • 사용자가 계정을 유지하는 기간 동안 보관
  • 사용자 요청 시 즉시 삭제
  • 계정 삭제 시 모든 관련 데이터 영구 삭제

4. 개인정보의 제3자 제공

본 서비스는 사용자의 개인정보를 제3자에게 판매, 대여, 양도하지 않습니다.

다음 경우에 한해 데이터가 외부 서비스와 연동됩니다:

  • Google Firebase: 데이터 저장을 위한 백엔드 서비스 (Google LLC 운영)
  • Google Drive API: 사용자가 Drive 저장을 선택한 경우, 해당 파일을 사용자 본인의 Drive에 업로드

위 경우 모두 사용자의 명시적 동의 하에 이루어지며, 데이터는 사용자 본인만 접근 가능합니다.

5. 사용자의 권리 및 행사 방법

사용자는 언제든지 다음 권리를 행사할 수 있습니다:

  • 조회: Google Firebase Console 또는 본 서비스 내에서 저장된 데이터 확인
  • 수정: 본 서비스의 편집 기능 사용
  • 삭제: 본 서비스 내에서 개별 항목 삭제 또는 전체 계정 삭제 요청
  • 권한 철회: Google 계정 보안 설정에서 KickClip 접근 권한 언제든지 취소 가능

계정 전체 삭제를 원하시면 아래 연락처로 요청해주시기 바랍니다.

6. 보안 조치

사용자 정보 보호를 위해 다음 조치를 적용합니다:

  • 모든 통신에 HTTPS 프로토콜 적용
  • OAuth 2.0 기반 안전한 인증
  • Firebase Security Rules를 통한 사용자별 데이터 접근 제어
  • 최소 권한 원칙: Drive 접근은 drive.file scope만 요청 (본 앱이 생성/접근한 파일에만 제한)
  • 민감한 데이터(예: 이메일 전체 내용, 비밀번호) 수집 안 함

7. 아동 보호

본 서비스는 만 13세 미만 아동을 대상으로 하지 않습니다. 만 13세 미만 아동의 개인정보가 수집된 사실이 확인되면 즉시 삭제합니다.

8. 개인정보처리방침의 변경

본 방침은 법령 및 서비스 변경에 따라 수정될 수 있습니다. 변경 시 본 페이지를 통해 공지하며, 중대한 변경 시 이메일 등을 통해 별도 안내드립니다.
